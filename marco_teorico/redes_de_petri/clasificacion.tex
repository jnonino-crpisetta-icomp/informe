\section{Clasificación de las Redes de Petri}
	\label{sec:clasificacion}

	Las \textbf{\emph{Redes de Petri (PN)}} se pueden considerar como autómatas formales y generadores de lenguajes formales. En las siguientes secciones, se analizarán un conjunto de sintaxis restrictivas que limitan el poder de modelado de las Redes de Petri pero permiten asegurar que el modelo generado cumple con ciertas propiedades.
	\begin{figure}[H]
		\centering
		\includegraphics[width=1\linewidth]{./marco_teorico/redes_de_petri/img/Petri08}
		\caption{Subclases de Redes de Petri}
		\label{fig:Petri08}
	\end{figure}
	
	\subsection{Grafos o máquinas de estado}

		Para representar una máquina de estados con una Red de Petri, se deben cumplir las siguientes condiciones: cada transición tiene un único arco de entrada y un único arco de salida, las plazas no tienen restricción y solo existe un token en toda la red. Esto significa que no puede existir concurrencia, pero si puede haber conflictos. Los conflictos se deben a que como una plaza puede tener muchos arcos de salida, no se puede determinar hacia donde irá un token. Por ejemplo, en la Figura \ref{fig:Petri09} un token en la plaza $P0$ generará conflicto entre las transiciones $T0$ y $T1$ dado que no se puede determinar cual debe ser disparada.
		\begin{figure}[H]
			\centering
			\includegraphics[width=.8\linewidth]{./marco_teorico/redes_de_petri/img/Petri09}
			\caption{Red de Petri restringida a máquina de estados}
			\label{fig:Petri09}
		\end{figure}
		
		Matemáticamente, sea $t$ una transición perteneciente al conjunto de transiciones $T$ se tiene que:
		\begin{equation}
			\forall t\in T : \mid t\bullet \mid =\mid\bullet t \mid =1
			\label{eq:petri_state_machine}
		\end{equation}
		
		\begin{figure}[H]
			\centering
			\includegraphics[width=0.25\linewidth]{./marco_teorico/redes_de_petri/img/Petri10}
			\caption{Ilustración de la restricción para formar máquinas de estados}
			\label{fig:Petri10}
		\end{figure}
		
		Generando un modelo con Redes de Petri que cumpla la restricción de la Ecuación \ref{eq:petri_state_machine} es posible garantizar que el modelo cumple con todas las propiedades propias de las máquinas de estado, es decir, es conservativa y acotada. La propiedad de vivacidad en este tipo de redes, es simple de demostrar y puede ser calculada de forma lineal con el algoritmo de Tarjan \cite{diaz_petri}.
	
	\subsection{Grafos de marcado}
	
		Para representar un grafo marcado con una Red de Petri debe cumplirse que: una plaza sólo puede tener un único arco de entrada y un único arco de salida, las transiciones no tienen restricciones. De esta manera, si es posible que exista concurrencia y además, no existen conflictos dado que un token solo puede disparar una transición.
	 
		Matemáticamente, sea $p$ una plaza perteneciente al conjunto de plazas $P$ se tiene que:
		\begin{equation}
			\forall p\in P : \mid p\bullet \mid =\mid\bullet p \mid =1
			\label{eq:petri_grafo_marcado}
		\end{equation}
		
		\begin{figure}[H]
			\centering
			\includegraphics[width=0.25\linewidth]{./marco_teorico/redes_de_petri/img/Petri11}
			\caption{Ilustración de la restricción para formar grafos de marcado.}
			\label{fig:Petri11}
		\end{figure}	
		
		Si la Red de Petri es de este tipo, la red es viva (sin deadlock) si y sólo si cualquier circuito elemental de la red incluye un lugar que inicialmente estaba marcado \cite{diaz_petri}. Una red de este tipo, será estructuralmente limitada si es fuertemente conectada \cite{diaz_petri}.
	
	\subsection{Redes de libre elección}
		
		En las redes de libre elección, todo arco que va desde una plaza hacia una transición, es el único arco que sale desde esa plaza o es el único que entra en esa transición.
		\begin{equation}
			\forall p\in P : (\mid p\bullet \mid\leq 1) \vee (\mid\bullet(p\bullet) \mid =\{1\}
			\label{eq:petri_libre_eleccion}
		\end{equation}

	\subsection{Redes de elección libre extendidas}
	
		Si dos lugares tienen alguna transición de salida común, entonces ellos tienen todas sus transiciones de salida en común, matemáticamente es equivalente a:
		\begin{equation}
			Sea\;p1\;y\;p2\in P: p1\bullet \cap p2\bullet \neq \phi \Rightarrow \mid p1\bullet \mid=\mid p1\bullet
			\mid
			\label{eq:petri_libre_extendida}
		\end{equation}
